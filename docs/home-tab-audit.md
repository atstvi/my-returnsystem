## 0. 새 레퍼런스 분석 — todomate (중간창/아이콘/색배치)

두 스크린샷에서 **복제가 아니라 원리**만 뽑는다(워크플로 규칙).

**(A) 바텀시트 액션 메뉴 — "리스트 메뉴"**
- 하단에서 올라오는 시트: 상단 **드래그 핸들**, 가운데 정렬 타이틀, 액션 목록, 하단 `취소 / 확인` 2버튼.
- 각 행 = **원형 컬러 아이콘 뱃지** + 라벨 + 우측 **선택 라디오**. 즉 "누르면 바로 실행"이 아니라
  **선택 → 확인**의 2단계(되돌리기 어려운 작업일수록 신중하게).
- 파괴적 액션(모든 할 일 삭제)은 **빨강 아이콘 + 빨강 텍스트**로 명확히 구분.
- 여백이 넉넉하고 구분선/박스가 거의 없음 — 아이콘·타이포·간격만으로 위계.

**(B) 설정 리스트**
- 아이콘(얇은 1.x px 아웃라인, 단색) + 라벨의 단순 행 나열. 박스·구분선 없음, 큰 여백.
- 아이콘 스타일이 **한 세트로 완전히 일관**됨(같은 굵기·같은 코너·같은 광학 크기).

**Return에 주는 시사점(원리)** — *오너 방향: 바텀시트 형식은 채택하지 않는다. 좋았던 건 형식이 아니라
**버튼 위치·모양잡힘·용도 적합성·전 창 통일감**이다. 중앙 모달을 유지하되 그 느낌을 준다.*
1. **버튼 멘탈모델 통일.** 취소(dismiss) = **왼쪽 아래**, 확인(commit) = **오른쪽 아래**. 딱 봤을 때
   어디를 누를지 예측 가능. → `.modal-foot`에 적용 완료.
2. **아이콘은 한 세트로.** 이미 24px·1.7 stroke SVG 라인아이콘 체계가 있는데 일부 중간창이 **텍스트
   글리프(✓ ✕ ↗, ◈)**를 써 이질감 → SVG로 교체(시그널 다이얼로그 적용 완료, 나머지 전수 추적).
3. **파괴적 작업은 스타일 통일.** native `confirm()`과 `openConfirmDialog()` 혼재 → 후자로 통일,
   `danger`/라벨을 용도에 맞게(fit-to-purpose). 영어 문구 한글화.
4. **용도에 맞는 창.** 확인창/편집창/선택창이 같은 뼈대(head·body·foot·버튼 위치)를 공유하되 용도에
   맞는 내용·강조색(파괴=빨강, 확정=액센트)을 갖게.

## 2. Phase 6 — 상태(States) 점검표 (홈)

각 컴포넌트가 아래를 다 갖췄는지 감사(❌=미흡):

- 오늘 할일: empty ❌(회색텍스트) · first-use ❌ · overflow ✅(스크롤) · zero-of-4 ❌
- 빠른 메모 위젯: memo empty ❌ · values empty ✅ · photo empty ❌ · music empty ❌ · loading ❌
- 시그널 리스트 모달: empty ✅(문구) · destructive confirm ⚠(혼재) · icon ❌(글리프)
- 타임블록: no-blocks empty ❌ · now-line ✅ · overflow ✅
- 습관: zero-habits empty ❌ · done feedback ✅
- 배너: no-image first-use ⚠(약함) · edit/position/sticker ✅
- 전역: offline/sync-fail ❌ · toast success ✅

→ **가장 큰 공백: 빈 상태·로딩·오프라인.** 홈은 "매일 처음 보는 화면"이라 빈/첫사용 상태 품질이
UX 체감을 좌우한다.

## 3. 우선순위 백로그 (구현 제안)

**P0 — 통일된 모달 시스템 + 일관성 (먼저) — ✅ 대부분 반영**
1. ✅ **버튼 배치 통일**(M4): `.modal-foot` — 취소(ghost) 왼쪽 아래 / 확인(primary·danger) 오른쪽
   아래. 단일 버튼은 우측 유지, 3버튼은 보조-좌 / 취소+확정-우. 전 모달 공통.
2. ✅ **버튼 시스템 단일화**: 스타일 없던 `.modal-btn`(지표 편집 모달) → `.btn .btn-ghost/.btn-primary`.
3. ✅ **확인창 fit-to-purpose**(M2): `openConfirmDialog(…, opts)` — `danger`/라벨을 용도에 맞게.
4. ✅ **홈 시그널 다이얼로그 아이콘**(M1·G1): `✓✕↗` → SVG 라인아이콘(의미별 색 유지).
5. ✅ **영어 문구 한글화**: 스티커 제거 확인 2건.
6. ⏭ **남은 글리프 전수 교체**(G1): `◈`(위젯 빈상태 아이콘) 등 — P1 빈상태 작업과 함께.
7. ⏭ **native `confirm()` → `openConfirmDialog` 수평 전개**(M2): 홈 외 탭 포함(콜백 구조 전환이라 사이트별
   신중히). 홈/버그성부터.

**P1 — 마찰 제거**
8. ✅ **오늘 할일 전(全)빈 상태**(C1): 사분면 4개 "비어 있음" → 단일 "오늘 잡아둔 할 일이 없어요" +
   할 일 추가 CTA. `◈` 위젯 빈상태 아이콘 → SVG.
9. ✅ **상황 카드 미리보기**(C6): 이미 `sit-msg`(가장 가까운 마감 문장) + `sit-upcoming`(상위 3건 제목)로
   충족 — 추가 작업 불필요로 판단.
10. ✅ **중간창 방향 확정**(M3·M4): 전부 중앙 모달 유지 + 통일 뼈대(§3.5). 바텀시트 불채택(오너 결정).

**P2 — 다듬기 / 기능 아이디어 (오너 결정 대기)**
> 아래는 "다듬기"가 아니라 **밀도 있는 UI에 정보/기능을 추가**하는 아이디어라, 폴리시가 아니라 별도
> 설계 판단이 필요하다. 리뷰 후 원하는 것만 진행 권장.
11. 습관 미니 스트릭(C3) — 습관 행에 주간 점 추가. 정보밀도↑지만 행이 이미 촘촘.
12. 타임블록 점프 힌트(C4) — 빈 그리드에서 가장 가까운 일정으로 점프.
13. 위젯 4종 상태 완전 통일(C2) — memo/photo/music 빈·로딩을 `.hw-empty` 하나로.
14. 다크모드 카드 분리 톤(G4) — 현재 elev-1 링으로 구분됨(라이트 테마 사용 중이라 우선순위 낮음).

**별건**: 알림 센터(G2).

**별건(제안서 필요)**
8. 알림 센터(G2) — 홈 시그널/마감/습관과 연동. 스코프가 커서 독립 설계.

## 4. Phase 8 — 자기비평 / 열린 결정 (오너 확인 필요)

- **바텀시트 불채택(오너 결정).** 중앙 모달 유지. todomate에서 취한 건 형식이 아니라 버튼 배치·통일감·
  용도 적합성 — §3.5에 통일 규칙으로 반영 완료.
- "선택→확인 2단계"(todomate)는 안전하지만 클릭이 늘어 원칙(불필요 클릭 제거)과 **부분 상충**. 되돌리기
  어려운 파괴적 작업에만 한정 적용 권장(현재는 openConfirmDialog 단일 확인으로 충분).
- 알림 기능은 아이디어 단계 — 데이터 소스(마감/처리 필요/습관)·트리거·저장(동기화 위험) 설계가 선행.
- 이 감사는 **나 탭 한정**. 인박스/할일/루틴에도 같은 중간창·아이콘·상태 원리가 그대로 적용 가능 —
  나 탭에서 패턴을 확정한 뒤 수평 전개 권장.
