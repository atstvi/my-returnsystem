# 충전(Recharge) 탭 재설계 노트

키워드: #생산성 #sns(x)st #mz20대여성st #세련 #깔끔 · 로즈 액센트/디자인 토큰 정합.

## 현황 (재설계 시작 시점)

- **충전 페이지**(`#page-recharge` → `.recharge-page`)
  - 히어로(💙 "괜찮아, 쉬어도 돼" + 시각)
  - 체크 카드(`.recharge-check-card` → `openRechargeCheck()`가 별도 `check` 페이지로 이동)
  - 에너지 카드(`.energy-card` / `.energy-levels`) — `goPage('recharge')` 시 `staminaMountEnergy()`가
    **공유 스태미나 슬라이더**(`stamina_level_v1`, 하루 0–100, 3밴드→routineCondition / 5밴드→recharge_energy)를
    마운트. `.energy-card-title`은 슬라이더가 뜨면 숨김. 사진 밴드(`stamina_photos_v1`)도 여기 반영.
  - 충전되는 것들 / 방전되는 것들(`#charge-items` / `#drain-items`) — `rcRenderList`, `+기록`(로그 패널),
    `편집`(`openActModal`).
  - 성찰(`.reflect-card`, `#reflect-ta`/`#reflect-list`, `saveReflect`) — "지금 진짜 필요한 게 뭐야?"
- **체크인 플로우**(별도 `check` 페이지, `#check-inner`) — 랜딩 → 4스텝(환경/상태/집중할 것/꺼내봐,
  `STEPS`) → 결과(`renderResult`: 내 입력 + AI 읽기 + 오늘의 다짐 → `saveResult`).

## 관찰된 개선점

1. **데스크탑 공간 낭비** — 좁은 1단 중앙 정렬(680px)이라 좌우가 텅 빔. 할일·타이머·루틴은 이미 구역화 풀블리드.
2. 에너지 UI가 충전 탭(스태미나 슬라이더)과 체크인 스텝(이모지 버튼)에서 다름 — 단, 슬라이더는 사진밴드
   기능이 있는 의도된 리치 컴포넌트라 함부로 통일하지 않음.
3. 충전 로그의 보상 뷰 부재(이번 주 충전 횟수/잔디), 에너지 추세 스파크라인 부재 — 후속 스코프.

## A. 데스크탑 구역화 (2단 레이아웃)

- `.recharge-body`를 폰=단일 컬럼 / 넓은 화면(≥880px)=2존 그리드로. **DOM/훅은 그대로**, 섹션들을
  `.rc-col`(좌·우) 래퍼로만 감쌈.
  - **좌(지금 나 상태)**: 체크 카드 · 에너지 슬라이더 · 성찰("지금 진짜 필요한 게 뭐야?")
  - **우(충전 활동)**: 충전되는 것들 · 방전되는 것들
- 각 `.rc-col`은 독립 flex 컬럼이라 높이가 독립(마조너리 느낌) — 그리드 행 정렬로 인한 빈틈 없음.
- 성찰을 좌측으로 옮겨 좌우 높이 균형 + "상태 확인→성찰→활동" 흐름. 폰에서는 체크→에너지→성찰→충전→방전.
- 히어로는 넓은 화면에서 상단 여백(`sp-16`→`sp-10`)만 줄여 두 컬럼이 접힘선 위로 오게.
- `max-width` 680→1080px, 좌우 패딩 `clamp(sp-6,4vw,sp-12)`.
- 스태미나 슬라이더·활동 로그·성찰 저장 로직 미변경. 제거된 컨트롤/훅 없음(8/8 유지). 가로 오버플로 0,
  라이트/다크/모바일 pageerror 0, `npm test` 32스위트 통과.
